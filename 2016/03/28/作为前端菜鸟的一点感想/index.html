作为前端菜鸟的一点感想

发表于 2016-03-28   |  阅读量 次   |  

接触前端是在大三，一门课需要写一个有选课功能的静态网页，于是在什么都不懂得情况下开始了前端之路。

PHP && HTML

作为什么都不懂但是作业又不得不写的渣渣，只能lean from google。但是不幸的是，我一开始就迈错了坑，竟然以为html只能和php写在一起，并且愚蠢地将所有的文件后缀一律改成了php。然后疯狂的在html代码里面穿插着诸如

的代码。当时觉得php无所不能，配合着html想怎么写怎么写，酷炫无比。最后作业顺利完成，并且还奇迹般在网页上加了一点css，顿时觉得自己逼格蹭蹭上涨，心里也有个声音告诉哥: 恭喜你入坑了。

因与果

世界上所有的事情都有因果，我之所以是我，是因为之前的我的行为造就了我–就如同一个崩坏的斐波拉切数列。之前的一节课上我与前端结缘，结下了我申请实习的果。显而易见，我选择了JD上写有web development的职位去申请，并且如愿以偿。

emacs && sublime text

我的运气一向很好，实习的时候遇到了一位有经验的，并且乐于帮助我的Team Leader – Mark。他教会了我很多，并且带我在前端的路上走了一程，我很感激他。他教我用emacs,但是我最终还是选择了sublime text，可能是因为我本身就不是那么极客的人，我更倾向于钻研怎么去砍树，而不是去自己做个斧头，而是打算直接买个电锯得了。sublime text已经满足了我所有的需求。emacs虽好，但是我并不愿意在它上面多花时间，而且我希望保持我的思维干净，不让自己的生活变得复杂。

lamp && xampp

从我用sublime text而不是emacs已经可以看出，我注定不是一个会去自己设置环境的人，所以理所应当，在一位师兄的介绍下，我选择了lamp或者xampp去做web开发。

JavaScript

说到web开发，我进公司的第一天，就知道我在原则性认识上已经出现了错误，我把php和html看成了互不可缺的伴侣，但其实两个没啥必然关系，而且程序员们也不会设计个非得把这两个绑定在一起的操蛋玩意儿。但是做web开发，可以说JavaScript确实是必不可少的。公司的Mark便普及了最基本的web知识给我，让我从基础上理解了js，html，css，php存在的意义。这也使我能够快速得在网上自主学习，了解了其他从这些基础引申出来的东西：less，scss,nodejs，react.js,vue.js,etc. 这个珍贵的学习过程给了我一个很大启发：如果可能，我希望学习任何一个领域的知识之前，都能找到一个业内人士，对我进行一个基础概念的普及。这个0到1的过程就像宇宙大爆炸的那一瞬间一样，短暂但及其有重量，它几乎决定了我之后在这个方面的学习兴趣以及效率。

感悟

说道感悟的正题，具体写代码的感悟不是没有，但我觉得不重要，这些所谓的方法论都是因人而异，而且在不同的阶段你的看法也会不同，我觉得只要跟着感觉走，你觉得这么写很好，很方便，那就继续。如果写出来的东西让你难受，那你就果断的删掉它，重新用适合自己的方式去写。我相信随着经验的增长，技能的熟悉，你自然会形成一套成熟有效的方法论。那我在乎的是什么方面的感悟呢？简单来说

  • 对自己：我们总是会自言自语，我们的想法和决定往往来源于你不断的自我拷问。你对自己的方式也就是能给别人看到的你的心态。所以我很看重这一方面的感悟。比如，我觉得自己要少对自己的行为有愧疚感，特别是涉及到钱，时间，面子。在心态上多一点利己主义，对一个总是处于弱势地位的人来说极其重要。再比如，人要学会自信，甚至有一些盲目的自信，狂妄总比自卑要好。不要让别人掌控到你，做自己人生的主宰者。
  • 对他人: 高中时期最大的体会是“宁杀君子莫惹小人”，而最近最大的体会则是“人之患，在于好为人师”。有时我会控制不住的喜欢去给别人言语上的提醒，我可能自我感觉我是为了他好，甚至让他占了便宜，觉得我辛苦总结出来的经验愿意分享与你，你竟然还不感激涕零。其实以一个旁观者的身份对别人的行为提出改进意见，它的目的是好的，但是太多太多是事情告诉我们，目的好不代表就做的对，做得对也不代表做得好，做得好更不代表做得妙。而且好为人师，最大就错在没有换位思考，或者可以说是太想当然的换位思考，以为别人需要你的帮助，会很感激你的意见。不是这样的，无知的人多，短视的人也多，但是自知的人很少。而自知的人不需要你的提醒，不自知的人更不需要你的提醒，甚至会因为你的善意而看低你。每个人都有自己的精神世界，所以不要轻易的发起一次世界大战。
